Configure cluster interface

Go to the “Clusters” tab, open the “Cluster master” link, hover over the row for the interface that is connected to the private network and select the edit icon.

  1. Set this interface to manage DHCP and DNS
  2. Set the “Router IP” to the default gateway

Fill in details for the dynamic and static ranges, remembering to leave gaps for the floating IPs.

  1. Dynamic range — that has as many IPs as there are total NICs connected to the network (minimum 15)
  2. Static range — that has as many IPs as there are machines connected to the network
  3. Floating IP range — that has as many IPs as instances that you’ll have in your cloud

Start enlisting machines!

Now you need to enlist and commission machines:

  1. Ensure all machines are set to PXE boot, if possible disable all other boot options, including local disk, in the BIOS
  2. Power the machines on so they will all appear in the “Nodes” tab of MAAS
  3. Alternatively, add machines or chassis by selecting the add hardware button
  4. Edit each machine filling in the power type and parameters
  5. Select all the machines and, using the “Take action” dropdown, “Commission” them.
  6. Wait until all machines have a “Ready” status
